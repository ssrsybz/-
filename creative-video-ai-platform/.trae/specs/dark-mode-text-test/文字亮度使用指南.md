# 深色模式文字亮度使用指南

## 1. 概述

本指南基于 WCAG 2.1 对比度标准、苹果 Human Interface Guidelines (HIG) 规范以及项目测试结果，为深色模式下的文字亮度设置提供明确的标准和使用建议。

## 2. 设计规范依据

### 2.1 WCAG 2.1 对比度标准

- **正常文本（小于 18pt 或 14pt 粗体）**：对比度至少为 4.5:1
- **大文本（18pt 或 14pt 粗体及以上）**：对比度至少为 3:1
- **UI 组件和图形**：对比度至少为 3:1

### 2.2 苹果 HIG 规范

- **深色模式背景**：推荐使用深灰色而非纯黑色，以减少眼睛疲劳
- **文字亮度**：根据文本重要性和层级设置不同的亮度值
- **可读性**：确保文字在不同屏幕亮度下都保持良好的可读性

## 3. 文字类型与推荐亮度值

### 3.1 主标题
- **亮度值**：98% 白色 (#FFFFFF)
- **对比度**：约 21:1（与深灰色背景）
- **适用场景**：页面标题、模态框标题
- **Tailwind CSS 类**：`text-white`

### 3.2 副标题
- **亮度值**：90% 白色 (#E6E6E6)
- **对比度**：约 15:1
- **适用场景**：section 标题、卡片标题
- **Tailwind CSS 类**：`text-gray-100`

### 3.3 正文文本
- **亮度值**：80% 白色 (#CCCCCC)
- **对比度**：约 10:1
- **适用场景**：主要内容、段落文本
- **Tailwind CSS 类**：`text-gray-200`

### 3.4 次要文本
- **亮度值**：60% 白色 (#999999)
- **对比度**：约 5:1
- **适用场景**：辅助信息、说明文本
- **Tailwind CSS 类**：`text-gray-300`

### 3.5 提示文本
- **亮度值**：45% 白色 (#737373)
- **对比度**：约 3:1
- **适用场景**：占位符、禁用状态文本
- **Tailwind CSS 类**：`text-gray-400`

### 3.6 链接文本
- **亮度值**：蓝色系 (#60A5FA)
- **对比度**：约 4.5:1
- **适用场景**：可点击链接、交互元素
- **Tailwind CSS 类**：`text-blue-400`

## 4. 背景颜色推荐

### 4.1 主背景
- **颜色值**：深灰色 (#1E293B)
- **Tailwind CSS 类**：`bg-slate-800`

### 4.2 卡片/面板背景
- **颜色值**：稍浅的深灰色 (#334155)
- **Tailwind CSS 类**：`bg-slate-700`

### 4.3 悬停/激活状态背景
- **颜色值**：更浅的灰色 (#475569)
- **Tailwind CSS 类**：`bg-slate-600`

## 5. 组件特定建议

### 5.1 按钮
- **主要按钮**：`bg-blue-600 text-white hover:bg-blue-700`
- **次要按钮**：`bg-slate-700 text-gray-200 hover:bg-slate-600`
- **文本按钮**：`text-blue-400 hover:text-blue-300`

### 5.2 输入框
- **背景**：`bg-slate-700`
- **边框**：`border-slate-600 focus:border-blue-500`
- **文本**：`text-gray-200`
- **占位符**：`placeholder:text-gray-400`

### 5.3 导航元素
- **活动状态**：`text-white bg-slate-700`
- **非活动状态**：`text-gray-300 hover:text-gray-200`

### 5.4 分割线
- **颜色**：`border-slate-700`

## 6. 实现示例

### 6.1 标题与正文

```jsx
// 主标题
<h1 className="text-2xl font-bold text-white">主标题</h1>

// 副标题
<h2 className="text-xl font-semibold text-gray-100">副标题</h2>

// 正文
<p className="text-gray-200">这是正文文本，应该使用 80% 白色的亮度。</p>

// 次要文本
<p className="text-gray-300">这是次要文本，用于辅助信息。</p>

// 提示文本
<p className="text-gray-400">这是提示文本，如占位符或禁用状态。</p>
```

### 6.2 按钮

```jsx
// 主要按钮
<button className="px-4 py-2 bg-blue-600 text-white rounded-lg hover:bg-blue-700">
  主要按钮
</button>

// 次要按钮
<button className="px-4 py-2 bg-slate-700 text-gray-200 rounded-lg hover:bg-slate-600">
  次要按钮
</button>

// 文本按钮
<button className="text-blue-400 hover:text-blue-300">
  文本按钮
</button>
```

### 6.3 卡片

```jsx
<div className="bg-slate-700 rounded-lg p-4">
  <h3 className="text-gray-100 font-semibold mb-2">卡片标题</h3>
  <p className="text-gray-200">卡片内容文本</p>
  <div className="mt-3 text-gray-300 text-sm">辅助信息</div>
</div>
```

## 7. 测试与验证

### 7.1 对比度测试
- 使用在线对比度检查工具验证文本与背景的对比度
- 确保所有文本元素都符合 WCAG 2.1 标准

### 7.2 视觉测试
- 在不同屏幕亮度下测试文字可读性
- 确保文字在深色背景下不会产生视觉疲劳
- 验证不同类型文本的层级关系清晰可辨

## 8. 最佳实践

1. **保持一致性**：在整个应用中使用统一的文字亮度标准
2. **层次分明**：通过亮度差异建立清晰的视觉层级
3. **按需调整**：根据具体场景和内容重要性调整文字亮度
4. **定期检查**：随着设计的演变，定期检查文字对比度是否符合标准
5. **用户反馈**：收集用户对深色模式可读性的反馈并进行优化
